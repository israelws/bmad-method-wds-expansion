## Purpose

Functional component specifications live in the project's `C-UX-Scenarios/Functional-Components/` folder. Page specifications reference these components instead of re-defining shared behavior inline.

This separation means:
- Components are defined once, used everywhere
- Changes propagate automatically
- Page specs stay focused on what's unique to that page
